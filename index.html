Comados Básicos de Git

Actividad del tutorial del git


Procesos

  1. working directory: Aquí es donde editamos el proyecto
  2. stating area: es donde escoemos que archivos está listos para pasar al repositorio.
  3. repository: aquí se registra todo lo que hicismos (repositorio)

Idetificarse en git

  1. git config --global user.email "email@email.com": Configura email del usuario
  2. git config --global user.name "name" : Asigna el nombre del usuario

  • git init: se utiliza para crear un proyecto nuevo en git.
  • git add file: se utiliza para pasar los archivos del working directory al staging area
  • git status: se usa para ver en que estado estan los archivos.
  • git commit: es para pasar del staging area al repository
  • git push: se usa para subir tus archivos a un repositorio remoto
  • git pull: te trae los cambios que han realizado los otros desarrolladores
  • git clone: copia desde el servidor central donde está el código a tu computador para comenzar a trabajar.

  • git log --> Muestra los commits que hemos creado
  • git checkout -- --> Revierte los cambios de los archivos modificados.
  • git diff --> muestra las diferencias hechas en los archivos
  • .gitignore --> archivo donde estarán los nombres de los archivos que el usuario quiere ignorar
